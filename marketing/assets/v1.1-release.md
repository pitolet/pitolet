# Pitolet v1.1

Pitolet can now bring an existing website onto the canvas.

Run the import command on the machine that can reach your site:

```sh
PITOLET_TOKEN=... pitolet import http://localhost:3000 \
  --to https://app.pitolet.com/w/your-workspace
```

The importer captures mobile, tablet, and desktop layouts and combines them into one responsive Pitolet frame. DOM that Pitolet understands stays editable. Images are copied into the asset store, while canvas, video, iframes, complex transforms, and other unsupported regions are kept as images.

Each import also saves source, imported, and difference images, with a similarity score for every captured width.

## Also in this release

- The same import API works in self-hosted Pitolet and Pitolet Cloud.
- Open self-hosted servers need no token; password-protected servers accept their shared password through `PITOLET_TOKEN`.
- Cloud imports use existing write-scoped agent tokens and workspace limits.
- Common form and table elements stay editable after import.
- Imported documents open directly from the URL returned by the CLI.
- The public site has a new product preview, clearer pricing, and shorter copy.

Running `pitolet` without a subcommand still starts the local server as before.

See the [README](https://github.com/pitolet/pitolet#readme) for every option and examples for authenticated pages.
