# Pitolet copy

Pitolet helps people fix interfaces made by coding agents. Lead with that problem. The browser, DOM, CSS, MCP, and code export are supporting facts.

## House rules

- Say what the user can do or see.
- Keep one point per sentence.
- Use plain verbs.
- Do not use em dashes.
- Do not make a list of three just to create rhythm.
- Avoid staged contrasts such as “not just X, but Y.”
- Cut throat-clearing phrases such as “it is worth noting.”
- Avoid stock words such as “seamless,” “unlock,” “robust,” and “revolutionary.”
- Do not claim that something is easy unless the page proves it.
- Read every sentence aloud before publishing it.

Factual lists are fine. A list of plan features or supported formats should contain the number of items the product actually has.
